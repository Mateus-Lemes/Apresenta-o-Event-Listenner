Event Listener

  • O que é um Event Listener e como adicioná-lo via JS?

    • É basicamente um conjunto de ações que são realizadas em um determinado elemento da página web, seja ele um texto, uma imagem, ou uma div, por exemplo. Muitas das interações do usuário que está visitando sua página com o conteúdo do seu site podem ser consideradas eventos.

    Para adicionar a sua aplicação web utlizamos da sintaxe element.addEventListener(event, function,useCapture)

  • Qual a diferença de um Event Listener de atributos HTML (como onclick)?

    • addEventListener

    addEventListener pode adicionar vários eventos a um elemento específico.

    addEventListener pode receber um terceiro argumento que pode controlar a propagação do evento.

    addEventListener só pode ser adicionado dentro de elementos script ou em arquivo JavaScript externo.

    addEventListener não funciona em versões anteriores do Internet explorer, que usa attachEvent.

    Atributos

    Atributo pode adicionar apenas um único evento a um elemento.

    A propagação de eventos não pode ser controlada por atributo.

    Atributo pode ser adicionado pelo HTML ou pelo JavaScript.

    onclick funciona em todos os navegadores.

  • O que são eventos de mouse e teclado em JavaScript?

    • Nada mais é do que eventos que ocorrem quando alguma ação é feita com o mouse ou teclado, seja ela clicar, passar por cima, pressionar uma tecla ou algo semelhante a isso.

    Exemplos mais utilizados de eventos de mouse e teclado:

    onBlur: remove o foco do elemento

    onChange: muda o valor do elemento

    onClick: o elemento é clicado pelo usuário

    onFocus: o elemento é focado

    onKeyPress: o usuário pressiona uma tecla sobre o elemento

    onLoad: carrega o elemento por completo

    onMouseOver: define ação quando o usuário passa o mouse sobre o elemento

    onMouseOut: define ação quando o usuário retira o mouse sobre o elemento

    onSubmit: define ação ao enviar um formulário

  • O que é uma função de callback (utilizada no Event Listener)?

    • A forma de utilização é parecida com a que já conhecemos. O segundo argumento da sintaxe recebe o que deve acontecer depois que o evento esperado (primeiro argumento) é recebido Nesse caso, o que deve acontecer será uma function, onde ela só vai ser chamada quando o evento também for chamado, ou seja, um callback.

  • O que é Event Bubbling em JS?

    • O Event bubbling ocorre quando um usuário interage com um elemento no HTML e o evento se propaga como “bolhas” por todos os elementos que estão aninhados a ele. Quando um evento “bubbling” é realizado sobre um determinado elemento, todos os ancestrais dele também serão acionados em ordem crescente de aninhamento até chegar ao último elemento
